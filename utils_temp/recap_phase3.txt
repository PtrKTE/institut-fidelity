2. CONTROLLERS
--------------------------------------------------------------------
app/Http/Controllers/Cliente/ClienteAuthController.php
  - showLogin / login          → Connexion cliente (email + mot_de_passe)
  - showVerifEmail / verifEmail → Verification email (active/inactive/absent)
  - showRegister / register    → Auto-inscription avec generation carte/barcode
  - showActivation / activate  → Activation compte via OTP 6 chiffres
  - resendOtp                  → Renvoi OTP (AJAX)
  - showResetRequest / sendResetOtp → Demande reset mot de passe
  - showResetConfirm / resetPassword → Confirmation reset avec OTP
  - logout                     → Deconnexion cliente

app/Http/Controllers/Cliente/EspaceClienteController.php
  - home         → Accueil (mini carte, stats, prochain RDV, top prestations)
  - carte        → Carte fidelite complete avec barcode + download PNG
  - profil       → Profil avec modification email/telephone via OTP
  - rendezvous   → Liste RDV + creation nouveau RDV + annulation
  - historique   → Liste factures paginee + graphique depenses
  - apiStats     → JSON: nb_visites, total_depense, taux_reduction, rdv_en_attente
  - apiHistorique → JSON: factures paginee avec prestations
  - apiDepensesMensuelles → JSON: depenses par mois (12 derniers)
  - apiTopPrestations → JSON: top 5 prestations de la cliente

4. ROUTES (28 routes, prefix /espace-cliente)
--------------------------------------------------------------------
Auth publiques :
  GET  /login              → Page connexion
  POST /login              → Traitement connexion
  GET  /verif-email        → Page verification email
  POST /verif-email        → API verification (JSON)
  GET  /inscription        → Page inscription
  POST /inscription        → API inscription (JSON)
  GET  /activation         → Page activation OTP
  POST /activation         → Traitement activation
  POST /resend-otp         → API renvoi OTP (JSON)
  GET  /reset-password     → Page demande reset
  POST /reset-password/send → API envoi OTP reset (JSON)
  GET  /reset-password/confirm → Page confirmation reset
  POST /reset-password/confirm → Traitement reset
  GET  /offline            → Page hors-ligne

Protegees (cliente_auth) :
  GET  /                   → Accueil / Home
  GET  /carte              → Carte fidelite
  GET  /profil             → Profil
  POST /profil/email       → API modification email (JSON)
  POST /profil/telephone   → API modification telephone (JSON)
  GET  /rendezvous         → Liste RDV
  POST /rendezvous         → API creation RDV (JSON)
  POST /rendezvous/{id}/annuler → API annulation RDV (JSON)
  GET  /historique          → Historique factures
  POST /logout              → Deconnexion
  GET  /api/stats           → API stats cliente (JSON)
  GET  /api/historique      → API historique pagine (JSON)
  GET  /api/depenses-mensuelles → API graphique depenses (JSON)
  GET  /api/top-prestations → API top 5 prestations (JSON)

6. FONCTIONNALITES PWA
--------------------------------------------------------------------
- Manifest installable (standalone, portrait, icones 192/512)
- Service Worker avec strategies :
  - Statiques (CSS/JS/fonts/images) : cache-first
  - API : network-first avec fallback JSON offline
  - Pages HTML : network-first avec fallback cache puis offline
- Banniere d'installation personnalisee (dismissible, localStorage)
- Badge compteur RDV en attente sur bottom nav
- Bottom navigation fixe (5 onglets : Accueil, Carte, RDV, Historique, Profil)
- Safe area padding pour iPhone (env(safe-area-inset-bottom))
- Responsive : max-width 480px (520px desktop), bottom nav centree sur desktop
